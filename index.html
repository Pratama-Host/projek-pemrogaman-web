🎯 6 Style Buttons

🎯 6 Style Buttons

📖 Apa itu Style Buttons?

Button adalah elemen paling sering diklik di website. Di sini ada 6 jenis button dengan karakteristik visual dan animasi yang berbeda, masing-masing punya "kepribadian" sesuai fungsinya.

Setiap button dibuat murni dengan CSS3 — tanpa gambar, tanpa library. Ini membuktikan bahwa CSS saja sudah bisa menghasilkan efek yang luar biasa.

1️⃣ Primary Button — Gradient + Hover Lift

Tombol utama halaman. Gradient merah-pink dengan efek "naik" saat di-hover menggunakan transform: translateY(-2px). Shadow makin tebal saat hover untuk memperkuat kesan terangkat.

CSS — Primary Button .btn-primary { background: linear-gradient(135deg, #ff6b6b, #fd79a8); box-shadow: 0 4px 15px rgba(255, 107, 107, 0.4); } .btn-primary:hover { transform: translateY(-2px); /* Naik 2px saat hover */ box-shadow: 0 6px 20px rgba(255, 107, 107, 0.6); }

2️⃣ Gradient Button — Scale Hover

Gradient biru-ungu. Saat di-hover tombol membesar 5% menggunakan transform: scale(1.05), memberi kesan tombol "maju" ke layar.

CSS — Gradient Button .btn-gradient { background: linear-gradient(135deg, #45b7d1, #a29bfe); color: white; } .btn-gradient:hover { transform: scale(1.05); /* Membesar 5% */ }

3️⃣ Outline Button — Fill on Hover

Awalnya transparan, hanya border. Saat di-hover, background terisi warna penuh. Teknik ini cocok untuk tombol alternatif seperti "Cancel" atau "Batal".

CSS — Outline Button .btn-outline { background: transparent; border: 3px solid #4ecdc4; color: #4ecdc4; } .btn-outline:hover { background: #4ecdc4; /* Background terisi saat hover */ color: white; }

4️⃣ 3D Button — Press Effect

Terlihat seperti tombol fisik. Shadow bawah menyimulasikan kedalaman. Saat diklik (:active), tombol "turun" dan shadow mengecil — persis seperti menekan tombol sungguhan.

CSS — 3D Button .btn-3d { background: #f9ca24; box-shadow: 0 6px 0 #d4a017; /* Shadow = kedalaman */ } .btn-3d:active { transform: translateY(4px); /* Turun saat ditekan */ box-shadow: 0 2px 0 #d4a017; /* Shadow mengecil */ }

5️⃣ Glow Button — Pulse Animation

Efek "cahaya berdenyut" yang jalan otomatis menggunakan @keyframes. Box-shadow berubah dari kecil ke besar berulang, menciptakan efek neon yang menarik perhatian.

CSS — Glow Button .btn-glow { background: #a29bfe; animation: glow 2s infinite; /* Loop tanpa henti */ } @keyframes glow { 0%, 100% { box-shadow: 0 0 20px rgba(162, 155, 254, 0.5); } 50% { box-shadow: 0 0 40px rgba(162, 155, 254, 0.8); } }

6️⃣ Animated Button — Moving Gradient

Background gradientnya dibuat 2x lebih lebar dari tombol, lalu posisinya digeser terus-menerus menggunakan background-position. Hasilnya adalah gradient yang seperti "bergerak" dari kiri ke kanan.

CSS — Animated Button .btn-animated { background: linear-gradient(90deg, #00b894, #4ecdc4, #00b894); background-size: 200% 100%; /* 2x lebih lebar */ animation: gradient-shift 3s ease infinite; } @keyframes gradient-shift { 0%, 100% { background-position: 0% 50%; } 50% { background-position: 100% 50%; } /* Geser ke kanan */ }

✅ Kesimpulan

  • Primary → Aksi terpenting (submit, beli, daftar)
  • Gradient → Fitur sekunder yang ingin ditonjolkan
  • Outline → Aksi alternatif/cancel yang tidak terlalu mencolok
  • 3D → Nuansa retro dan fun, cocok untuk game UI
  • Glow → Menarik perhatian ke fitur premium atau notifikasi
  • Animated → CTA (Call-to-Action) yang ingin sangat menonjol

🌓 Theme Toggle

Beralih antara Light Mode dan Dark Mode dengan satu klik

🌓 Theme Toggle

Light Dark

📖 Apa itu Theme Toggle?

Theme Toggle atau Dark Mode Switch adalah fitur yang memungkinkan pengguna memilih tampilan terang (light) atau gelap (dark) sesuai preferensi. Fitur ini kini menjadi standar di hampir semua aplikasi modern — Instagram, YouTube, Twitter, semuanya punya!

Komponen ini terdiri dari toggle switch CSS yang elegan dan CSS Variables yang secara instan mengubah seluruh palet warna halaman.

🎨 Cara Kerja: CSS Variables

Seluruh warna halaman disimpan dalam variabel CSS di :root. Saat class dark-theme ditambahkan ke <body>, variabel-variabel itu di-override oleh nilai dark mode — dan semua elemen yang menggunakan variabel itu ikut berubah secara otomatis!

CSS — Dual Theme Variables /* Light Mode (default) */ :root { --bg-primary: #f0f4ff; --bg-secondary: #ffffff; --text-primary: #2d3748; } /* Dark Mode (aktif saat body punya class dark-theme) */ body.dark-theme { --bg-primary: #1a202c; --bg-secondary: #2d3748; --text-primary: #f7fafc; } /* Elemen tinggal pakai variabelnya */ body { background: var(--bg-primary); color: var(--text-primary); transition: background 0.3s, color 0.3s; /* Animasi smooth */ }

🔘 Toggle Switch — Murni CSS

Switch berbentuk pil oval ini dibuat sepenuhnya dengan CSS. Elemen <input type="checkbox"> disembunyikan, lalu .toggle-slider menggantikan tampilannya. Lingkaran putih di dalamnya bergerak menggunakan transform: translateX().

CSS — Toggle Switch .toggle-switch { position: relative; width: 60px; height: 30px; display: inline-block; } .toggle-switch input { opacity: 0; } /* Sembunyikan checkbox asli */ .toggle-slider { position: absolute; inset: 0; background: linear-gradient(135deg, #f9ca24, #ff8c00); border-radius: 30px; cursor: pointer; } .toggle-slider:before { content: ""; position: absolute; width: 22px; height: 22px; left: 4px; bottom: 4px; background: white; border-radius: 50%; transition: 0.4s; } input:checked + .toggle-slider:before { transform: translateX(30px); /* Geser ke kanan */ }
JavaScript — Toggle Theme const themeToggle = document.getElementById('themeToggle'); themeToggle.addEventListener('change', () => { document.body.classList.toggle('dark-theme'); // Saat checkbox dicentang → class 'dark-theme' ditambahkan // Saat checkbox tidak dicentang → class 'dark-theme' dihapus });

✅ Kesimpulan

  • Gunakan CSS Variables agar tema bisa berubah hanya dengan satu class
  • Tambahkan transition pada body agar pergantian tema terasa mulus
  • Toggle switch bisa dibuat sepenuhnya dengan CSS tanpa gambar apapun
  • Dark mode bukan sekadar estetika — juga hemat baterai dan lebih nyaman di mata

📋 Accordion Panel

Konten yang bisa dibuka dan ditutup dengan animasi smooth

📋 Accordion Panel

HTML (HyperText Markup Language) adalah bahasa markup standar untuk membuat halaman web.

CSS (Cascading Style Sheets) digunakan untuk styling dan layout halaman web.

JavaScript adalah bahasa pemrograman yang membuat halaman web menjadi interaktif.

📖 Apa itu Accordion?

Accordion adalah komponen UI yang menyembunyikan konten di balik header. Saat header diklik, konten "membuka" ke bawah — seperti alat musik akordeon yang bisa dilipat dan dibuka.

Accordion sangat berguna untuk FAQ, Terms & Conditions, panel settings, atau konten apapun yang panjang namun perlu disajikan secara ringkas.

🏗️ Struktur HTML

Setiap item accordion terdiri dari .accordion-header (yang diklik) dan .accordion-content (yang tersembunyi). Icon + akan berputar 45° menjadi × saat terbuka.

HTML — Accordion Structure <div class="accordion-item"> <button class="accordion-header"> <span>Pertanyaan di sini?</span> <span class="accordion-icon">+</span> </button> <div class="accordion-content"> <p>Jawaban atau konten tersembunyi di sini.</p> </div> </div>

🎞️ Trik Animasi: max-height

Animasi "slide down" tidak bisa menggunakan height: auto karena CSS tidak bisa mentransisikan nilai auto. Solusinya: gunakan max-height dengan nilai yang cukup besar saat aktif, dan 0 saat tertutup.

CSS — Accordion Animation Trick /* Awalnya tertutup */ .accordion-content { max-height: 0; overflow: hidden; transition: max-height 0.3s ease; } /* Saat item aktif (dibuka) */ .accordion-item.active .accordion-content { max-height: 500px; /* Nilai lebih besar dari konten */ padding: 15px 20px; } /* Icon + berputar menjadi × */ .accordion-icon { transition: transform 0.3s; } .accordion-item.active .accordion-icon { transform: rotate(45deg); }
JavaScript — Toggle Accordion (One at a Time) const accordionHeaders = document.querySelectorAll('.accordion-header'); accordionHeaders.forEach(header => { header.addEventListener('click', () => { const item = header.parentElement; const isActive = item.classList.contains('active'); // Tutup semua accordion dulu document.querySelectorAll('.accordion-item').forEach(i => { i.classList.remove('active'); }); // Buka yang diklik (kecuali kalau sudah aktif = tutup) if (!isActive) { item.classList.add('active'); } }); });

✅ Kesimpulan

  • Gunakan max-height bukan height untuk animasi slide yang smooth
  • Logika "tutup semua, baru buka yang diklik" memastikan hanya satu item yang terbuka
  • Rotasi icon + → × memberikan feedback visual yang jelas
  • Accordion ideal untuk FAQ, spesifikasi produk, dan konten panjang yang perlu diringkas

🪟 Modal Dialog

Jendela popup yang muncul di atas konten utama untuk interaksi fokus

🪟 Modal Dialog

×

🎉 Selamat Datang!

Ini adalah contoh modal dialog yang interaktif. Klik tombol X atau area luar untuk menutup.

📖 Apa itu Modal Dialog?

Modal adalah jendela "pop-up" yang muncul di atas konten halaman, memblokir interaksi dengan konten di belakangnya sampai modal ditutup. Modal digunakan untuk konfirmasi, pesan penting, form input, atau preview gambar.

Komponen ini terdiri dari backdrop gelap (overlay) dan kotak konten di tengahnya, lengkap dengan animasi muncul yang smooth.

🏗️ Struktur HTML

Modal memiliki dua lapisan: .modal sebagai backdrop fullscreen, dan .modal-content sebagai kotak dialog yang sesungguhnya. Tombol .modal-close (×) untuk menutupnya.

HTML — Modal Structure <!-- Tombol untuk membuka modal --> <button id="openModal">Buka Modal</button> <!-- Modal: backdrop + content box --> <div class="modal" id="modal"> <div class="modal-content"> <span class="modal-close">&times;</span> <h3>Judul Modal</h3> <p>Isi konten modal di sini.</p> <button>Tutup</button> </div> </div>

🎞️ Animasi Modal dengan CSS

Modal awalnya display: none dan opacity: 0. Saat class active ditambahkan, modal muncul dengan fade-in dan kotak kontennya turun dari atas menggunakan transform: translateY().

CSS — Modal Backdrop & Animation /* Backdrop fullscreen */ .modal { display: none; position: fixed; inset: 0; /* top:0 right:0 bottom:0 left:0 */ background: rgba(0, 0, 0, 0.7); z-index: 1000; justify-content: center; align-items: center; } .modal.active { display: flex; /* Muncul saat active */ animation: modalFadeIn 0.3s ease; } /* Kotak dialog */ .modal-content { background: white; border-radius: 15px; padding: 30px; max-width: 500px; width: 90%; animation: modalSlideIn 0.3s ease; } @keyframes modalSlideIn { from { transform: translateY(-30px); opacity: 0; } to { transform: translateY(0); opacity: 1; } }
JavaScript — Open & Close Modal const modal = document.getElementById('modal'); const openModalBtn = document.getElementById('openModal'); const closeModal = document.querySelector('.modal-close'); // Buka modal openModalBtn.addEventListener('click', () => { modal.classList.add('active'); }); // Tutup via tombol × closeModal.addEventListener('click', () => { modal.classList.remove('active'); }); // Tutup saat klik backdrop (luar kotak) window.addEventListener('click', (e) => { if (e.target === modal) { modal.classList.remove('active'); } });

✅ Kesimpulan

  • Modal terdiri dari dua lapis: backdrop gelap dan kotak konten
  • Klik di luar kotak (pada backdrop) harus menutup modal — ini UX yang baik
  • Animasi slide down + fade in membuat modal terasa lebih halus dan profesional
  • Gunakan modal untuk: konfirmasi hapus, preview gambar, form login/register, atau pesan penting

Navigasi konten berlapis dengan tampilan satu panel aktif dalam satu waktu

📑 Tabs Navigation

Home Content

Selamat datang di halaman Home! Ini adalah konten tab pertama.

Profile Content

Informasi profil Anda akan ditampilkan di sini.

Settings Content

Pengaturan aplikasi tersedia di tab ini.

📖 Apa itu Tabs Navigation?

Tabs memungkinkan pengelompokan konten berbeda dalam satu area tampilan. Hanya satu panel (tab pane) yang terlihat dalam satu waktu, dan user bisa beralih antar konten dengan mengklik tombol tab.

Pola ini sangat umum di halaman profil, dashboard aplikasi, dan pengaturan — karena menghemat ruang tanpa mengorbankan aksesibilitas konten.

🏗️ Struktur HTML

Ada dua bagian utama: .tab-buttons berisi tombol-tombol tab, dan .tab-content berisi panel konten (.tab-pane) yang hanya satu bisa aktif sekaligus. Koneksi antara tombol dan panel menggunakan atribut data-tab.

HTML — Tabs Structure <div class="tabs"> <!-- Tombol tab --> <div class="tab-buttons"> <button class="tab-btn active" data-tab="tab1">🏠 Home</button> <button class="tab-btn" data-tab="tab2">👤 Profile</button> <button class="tab-btn" data-tab="tab3">⚙️ Settings</button> </div> <!-- Panel konten --> <div class="tab-content"> <div class="tab-pane active" id="tab1">Konten Home</div> <div class="tab-pane" id="tab2">Konten Profile</div> <div class="tab-pane" id="tab3">Konten Settings</div> </div> </div>

🎨 CSS: Tampilkan Hanya Panel Aktif

Semua .tab-pane disembunyikan dengan display: none secara default. Hanya yang memiliki class active yang akan ditampilkan. Tombol aktif mendapat style khusus sebagai indikator visual.

CSS — Show/Hide Tab Panes /* Sembunyikan semua panel */ .tab-pane { display: none; padding: 20px; } /* Tampilkan hanya yang aktif */ .tab-pane.active { display: block; animation: fadeIn 0.3s ease; } /* Tombol tab aktif */ .tab-btn.active { background: linear-gradient(135deg, var(--color-3), var(--color-5)); color: white; }
JavaScript — Tab Switching Logic const tabButtons = document.querySelectorAll('.tab-btn'); const tabPanes = document.querySelectorAll('.tab-pane'); tabButtons.forEach(btn => { btn.addEventListener('click', () => { const targetTab = btn.getAttribute('data-tab'); // Hapus 'active' dari semua tombol dan panel tabButtons.forEach(b => b.classList.remove('active')); tabPanes.forEach(p => p.classList.remove('active')); // Tambah 'active' ke tombol yang diklik dan panel yang sesuai btn.classList.add('active'); document.getElementById(targetTab).classList.add('active'); }); });

✅ Kesimpulan

  • Hubungan tombol dan panel menggunakan atribut data-tab sebagai key yang cocok dengan id panel
  • Pola "hapus semua, tambah ke yang aktif" adalah cara paling bersih untuk switching
  • Tambahkan animasi fadeIn pada panel saat muncul agar transisi terasa halus
  • Tabs cocok untuk: halaman profil, pengaturan, dashboard, detail produk dengan banyak seksi

🎚️ Slider Range

Input berbentuk track geser untuk memilih nilai dalam rentang tertentu

🎚️ Slider Range

📖 Apa itu Slider Range?

Slider Range adalah input HTML bertipe range yang memungkinkan user memilih nilai numerik dengan cara menggeser "thumb" (tombol bulat) di sepanjang track. Sangat intuitif untuk mengatur volume, brightness, harga, ukuran, dan sebagainya.

Di sini slider dikustomisasi penuh dengan CSS agar tampilannya sesuai tema — karena tampilan default browser sangat membosankan!

🏗️ HTML — Input Range Dasar

Atribut min, max, dan value mendefinisikan rentang dan nilai awal. Label di atas slider menampilkan nilai saat ini secara real-time melalui JavaScript.

HTML — Slider Range <div class="slider-container"> <label>Volume: <span id="sliderValue">50</span>%</label> <input type="range" class="slider" id="rangeSlider" min="0" <!-- Nilai minimum --> max="100" <!-- Nilai maksimum --> value="50" <!-- Nilai awal --> > </div>

🎨 Kustomisasi Tampilan Slider dengan CSS

Browser memiliki tampilan slider default yang berbeda-beda. Untuk konsistensi dan estetika, kita override menggunakan pseudo-element ::-webkit-slider-thumb dan ::-webkit-slider-runnable-track.

CSS — Custom Slider Appearance .slider { -webkit-appearance: none; /* Hapus tampilan default */ appearance: none; width: 100%; height: 8px; border-radius: 5px; background: linear-gradient(90deg, var(--color-2), var(--color-3)); outline: none; cursor: pointer; } /* Tombol bulat (thumb) */ .slider::-webkit-slider-thumb { -webkit-appearance: none; width: 24px; height: 24px; border-radius: 50%; background: white; box-shadow: 0 2px 8px rgba(0,0,0,0.3); cursor: grab; transition: transform 0.2s; } .slider::-webkit-slider-thumb:active { cursor: grabbing; transform: scale(1.2); }
JavaScript — Real-time Value Update const rangeSlider = document.getElementById('rangeSlider'); const sliderValue = document.getElementById('sliderValue'); // Event 'input' terpanggil setiap kali nilai berubah (saat digeser) rangeSlider.addEventListener('input', () => { sliderValue.textContent = rangeSlider.value; // rangeSlider.value selalu berisi nilai terkini (0-100) });

✅ Kesimpulan

  • Gunakan event 'input' (bukan 'change') agar nilai update real-time saat digeser
  • Hapus tampilan default dengan -webkit-appearance: none sebelum mengkustomisasi
  • Slider cocok untuk: volume, brightness, filter harga, progress pembelajaran, rating
  • Selalu tampilkan nilai numerik di samping slider agar user tahu posisi tepatnya

📊 Progress Bar

Visualisasi progres tugas atau loading yang bergerak secara dinamis

📊 Progress Bar

📖 Apa itu Progress Bar?

Progress Bar adalah elemen visual yang menunjukkan seberapa jauh sebuah proses telah berjalan — upload file, loading halaman, penyelesaian kuis, atau langkah-langkah pendaftaran. Bar ini sangat penting untuk memberikan feedback kepada user bahwa sesuatu sedang berjalan.

Di sini, progress bar dianimasikan menggunakan setInterval JavaScript yang secara bertahap memperlebar elemen fill dari 0% hingga 100%.

🏗️ Struktur HTML: Dua Lapis

Progress bar terdiri dari dua elemen bertumpuk: .progress-bar sebagai container (rel/latar), dan .progress-fill sebagai bar isian yang lebarnya diubah secara dinamis via JavaScript.

HTML — Progress Bar Structure <div class="progress-bar"> <!-- Fill: lebar diubah via JS, mulai dari 0% --> <div class="progress-fill" id="progressFill" style="width: 0%"></div> </div> <button id="startProgress">Start Progress</button>

🎨 CSS: Container dan Fill

Container memiliki border-radius dan background abu-abu. Fill memiliki gradient warna-warni dan transition agar perubahan lebarnya terasa smooth (bukan loncat).

CSS — Progress Bar Styling /* Container (latar abu-abu) */ .progress-bar { width: 100%; height: 20px; background: var(--bg-primary); border-radius: 10px; overflow: hidden; /* Penting agar fill tidak keluar */ box-shadow: inset 0 2px 5px rgba(0,0,0,0.1); margin-bottom: 20px; } /* Fill (bar berwarna) */ .progress-fill { height: 100%; background: linear-gradient(90deg, var(--color-2), var(--color-3), var(--color-5)); border-radius: 10px; transition: width 0.05s linear; /* Animasi smooth tiap increment */ }
JavaScript — Animated Progress dengan setInterval const startProgressBtn = document.getElementById('startProgress'); const progressFill = document.getElementById('progressFill'); startProgressBtn.addEventListener('click', () => { let progress = 0; progressFill.style.width = '0%'; // Reset ke 0 // setInterval: jalankan fungsi setiap X milidetik const interval = setInterval(() => { progress += 1; progressFill.style.width = progress + '%'; // Tambah 1% setiap 20ms if (progress >= 100) { clearInterval(interval); // Hentikan saat mencapai 100% } }, 20); // 20ms × 100 langkah = ~2 detik total });

✅ Kesimpulan

  • Struktur dua lapis (container + fill) adalah pola standar progress bar di CSS
  • overflow: hidden pada container wajib agar fill tidak meluber
  • Gunakan setInterval + clearInterval untuk animasi berbasis waktu yang terkontrol
  • Progress bar ideal untuk: upload/download, loading, onboarding steps, skill chart, kuis

📖 Tiga Gaya Card Efek

Card adalah unit konten yang paling sering digunakan di web modern — produk di e-commerce, artikel di blog, profil tim, dll. Di sini ada tiga variasi efek interaktif yang bisa membuat card terasa hidup: Hover Lift, Flip 3D, dan Glow Effect.

1️⃣ Hover Card — Lift & Shadow

Card naik ke atas dan shadow-nya membesar saat di-hover, menciptakan ilusi card "terangkat" dari permukaan. Efek paling umum dan aman digunakan di hampir semua konteks.

CSS — Hover Lift Effect .card-hover { transition: transform 0.3s, box-shadow 0.3s; } .card-hover:hover { transform: translateY(-8px); /* Naik 8px */ box-shadow: 0 15px 35px rgba(0, 0, 0, 0.15); /* Shadow membesar */ }

2️⃣ Flip Card — CSS 3D Transform

Card memiliki dua sisi (depan & belakang). Saat diklik, card berotasi 180° di sumbu Y menggunakan CSS 3D Transforms. Ini salah satu efek CSS paling dramatis yang selalu memukau!

CSS — Flip Card 3D /* Container: buat ruang 3D */ .card-flip { perspective: 1000px; /* Jarak pandang 3D */ cursor: pointer; } /* Inner yang berotasi */ .card-inner { position: relative; transform-style: preserve-3d; /* Children jadi 3D */ transition: transform 0.6s; } /* Saat card.flipped ditambahkan */ .card-flip.flipped .card-inner { transform: rotateY(180deg); } /* Sisi depan dan belakang */ .card-front, .card-back { position: absolute; inset: 0; backface-visibility: hidden; /* Sembunyikan sisi yang tidak dilihat */ } .card-back { transform: rotateY(180deg); /* Awalnya terbalik */ }
JavaScript — Toggle Flip const flipCards = document.querySelectorAll('.card-flip'); flipCards.forEach(card => { card.addEventListener('click', () => { card.classList.toggle('flipped'); // Toggle: tambah 'flipped' jika belum ada, hapus jika sudah ada }); });

3️⃣ Glow Card — Animated Box Shadow

Card memancarkan "cahaya" yang berdenyut menggunakan animasi @keyframes pada box-shadow. Efek ini sangat cocok untuk card fitur unggulan atau promo spesial.

CSS — Glow Effect Animation .card-glow { animation: cardGlow 2.5s ease-in-out infinite; } @keyframes cardGlow { 0%, 100% { box-shadow: 0 0 15px rgba(162, 155, 254, 0.3); } 50% { box-shadow: 0 0 35px rgba(162, 155, 254, 0.7), 0 0 60px rgba(78, 205, 196, 0.3); } }

✅ Kesimpulan

  • Hover Lift: Efek universal, cocok untuk grid produk, artikel, atau profil
  • Flip 3D: Butuh perspective + preserve-3d + backface-visibility: hidden — ketiganya wajib ada!
  • Glow: Menarik perhatian, tapi jangan dipakai berlebihan agar tidak mengalihkan fokus
  • Card yang baik selalu memiliki feedback visual saat interaksi (hover/klik)

📖 Dropdown + 4 Style Galeri

Dropdown adalah menu pilihan yang tersembunyi dan hanya muncul saat tombol diklik. Yang membuat bagian ini istimewa: setiap pilihan menampilkan galeri gambar dengan gaya interaksi yang berbeda-beda — total 4 teknik UI sekaligus dalam satu komponen!

Ini adalah cara terbaik untuk belajar berbagai teknik CSS dan JavaScript secara kontekstual.

🔽 Cara Kerja Dropdown

Tombol toggle memunculkan/menyembunyikan .dropdown-menu dengan class active. Klik di luar area dropdown (event di document) menutupnya kembali secara otomatis.

JavaScript — Dropdown Toggle Logic const dropdownToggle = document.querySelector('.dropdown-toggle'); const dropdownMenu = document.querySelector('.dropdown-menu'); // Buka/tutup saat tombol diklik dropdownToggle.addEventListener('click', (e) => { e.stopPropagation(); // Cegah event bubble ke document dropdownMenu.classList.toggle('active'); }); // Tutup saat klik di mana saja di luar dropdown document.addEventListener('click', () => { dropdownMenu.classList.remove('active'); });

1️⃣ Dashboard Style — Hover Overlay

Gambar ditampilkan dalam grid. Saat di-hover, overlay gelap muncul dari bawah menggunakan transform: translateY(). Teknik ini sering dipakai di Instagram dan Pinterest.

CSS — Hover Overlay .gallery-overlay { transform: translateY(100%); /* Sembunyi di bawah */ transition: transform 0.3s; } .gallery-item:hover .gallery-overlay { transform: translateY(0); /* Muncul ke atas saat hover */ }

2️⃣ Profile Style — Card dengan Tombol

Gambar ditampilkan layaknya card profil dengan nama dan tombol "View Details". Pola ini umum di halaman team/staff dan listing produk.

3️⃣ Settings Style — Flip Card 3D

Setiap gambar bisa di-flip seperti flashcard menggunakan CSS 3D Transforms (konsep yang sama dengan Card Effects di atas).

4️⃣ Logout Style — Lightbox / Zoom Modal

Klik gambar → gambar tampil fullscreen di modal gelap. Ini disebut lightbox, pola yang sangat umum di galeri foto, portofolio, dan e-commerce.

CSS — Zoom Modal (Lightbox) .zoom-modal { position: fixed; inset: 0; background: rgba(0, 0, 0, 0.95); /* Backdrop hampir hitam */ display: flex; justify-content: center; align-items: center; } .zoom-content { max-width: 90%; max-height: 90%; animation: zoomIn 0.3s ease; } @keyframes zoomIn { from { transform: scale(0.7); opacity: 0; } to { transform: scale(1); opacity: 1; } }

✅ Kesimpulan

  • Dropdown memanfaatkan pola toggle class active + event stopPropagation agar tidak langsung tertutup saat dibuka
  • Keempat style galeri menunjukkan bahwa data yang sama bisa dipresentasikan dengan cara yang sangat berbeda
  • Pilih style galeri sesuai konteks: overlay → media sosial, card → profil, flip → edukasi, lightbox → portofolio

🍞 Toast Notification

Pesan notifikasi singkat yang muncul sebentar lalu menghilang otomatis

🍞 Toast Notification

📖 Apa itu Toast Notification?

Toast adalah notifikasi kecil yang muncul sebentar (biasanya 3 detik) di pojok layar, lalu menghilang sendiri tanpa perlu interaksi dari user. Namanya dari roti yang "pop up" dari toaster!

Toast ideal untuk memberi feedback singkat: "berhasil disimpan", "koneksi terputus", "pesan terkirim", dll — tanpa mengganggu alur kerja user seperti modal.

🎞️ Teknik Animasi: Slide In dari Kanan

Toast awalnya di luar layar (translateX(400px)) dan transparan. Saat class show ditambahkan, toast bergerak masuk ke posisi normal. Setelah 3 detik, class show dihapus dan toast kembali ke posisi awal (slide out).

CSS — Toast Slide In/Out Animation /* Container di pojok kanan atas */ #toastContainer { position: fixed; top: 20px; right: 20px; z-index: 2000; } /* State awal: tersembunyi di kanan */ .toast { opacity: 0; transform: translateX(400px); transition: all 0.3s ease; padding: 15px 20px; border-radius: 10px; color: white; min-width: 300px; margin-bottom: 10px; } /* State aktif: muncul */ .toast.show { opacity: 1; transform: translateX(0); } /* Tipe-tipe toast */ .toast-success { background: linear-gradient(135deg, #00b894, #4ecdc4); } .toast-info { background: linear-gradient(135deg, #45b7d1, #a29bfe); } .toast-warning { background: linear-gradient(135deg, #f9ca24, #ff7675); } .toast-error { background: linear-gradient(135deg, #ff6b6b, #fd79a8); }
JavaScript — showToast() Function function showToast(type) { const container = document.getElementById('toastContainer'); const toast = document.createElement('div'); toast.className = `toast toast-${type}`; // Tentukan warna berdasarkan tipe const messages = { success: '✅ Berhasil! Operasi selesai.', info: 'ℹ️ Info: Ada informasi untuk Anda.', warning: '⚠️ Peringatan: Perhatikan ini.', error: '❌ Error: Terjadi kesalahan.' }; toast.textContent = messages[type]; container.appendChild(toast); // Tunggu 10ms lalu tambahkan class 'show' (agar transisi berjalan) setTimeout(() => toast.classList.add('show'), 10); // Setelah 3 detik, mulai animasi keluar setTimeout(() => { toast.classList.remove('show'); // Hapus dari DOM setelah animasi selesai (300ms) setTimeout(() => toast.remove(), 300); }, 3000); }

✅ Kesimpulan

  • Toast dibuat secara dinamis dengan createElement agar bisa muncul banyak sekaligus
  • Delay 10ms antara appendChild dan classList.add('show') diperlukan agar CSS transition terpicu
  • Selalu hapus elemen toast dari DOM setelah animasi keluar selesai agar tidak menumpuk
  • Gunakan 4 warna berbeda (success/info/warning/error) agar user langsung paham jenis notifikasinya

⏳ Loading Spinners

Empat jenis animasi loading yang dibuat sepenuhnya dengan CSS

⏳ Loading Spinners

📖 Apa itu Loading Spinner?

Loading spinner adalah animasi visual yang memberi tahu user bahwa sesuatu sedang diproses — konten sedang dimuat, data sedang dikirim, dll. Tanpa spinner, user tidak tahu apakah website-nya hang atau masih bekerja.

Keempat spinner di sini dibuat 100% murni CSS menggunakan kombinasi border, border-radius, dan animasi @keyframes — tidak ada gambar GIF sama sekali!

1️⃣ Spinner Klasik — Border Rotate

Teknik paling populer: lingkaran penuh dengan satu bagian border berwarna. Keseluruhan elemen diputar dengan animation: spin. Border yang "hilang" memberi ilusi yang berputar.

CSS — Classic Spinner .spinner-1 { width: 60px; height: 60px; border: 6px solid var(--bg-primary); /* Border abu-abu */ border-top: 6px solid var(--color-1); /* Hanya atas yang berwarna */ border-radius: 50%; /* Bentuk lingkaran */ animation: spin 1s linear infinite; /* Putar terus */ } @keyframes spin { from { transform: rotate(0deg); } to { transform: rotate(360deg); } }

2️⃣ Dual Color Spinner — Multi-border

Dua warna sekaligus! Border transparan di sisi kiri-bawah, berwarna di kanan-atas. Menggunakan dua properti border berbeda agar efeknya terlihat dinamis dan modern.

CSS — Dual Color Spinner .spinner-2 { border: 6px solid transparent; /* Semua transparan */ border-top: 6px solid var(--color-2); /* Atas: teal */ border-right: 6px solid var(--color-3); /* Kanan: biru */ border-radius: 50%; animation: spin 0.8s linear infinite; /* Lebih cepat */ }

3️⃣ Pulse Spinner — Scale Animation

Berbentuk kotak yang berdenyut membesar-mengecil. Menggunakan transform: scale() yang beranimasi bolak-balik. Berbeda dari spinner putar — ini lebih cocok untuk state "menunggu" yang santai.

CSS — Pulse Spinner .spinner-3 { background: linear-gradient(135deg, var(--color-5), var(--color-3)); border-radius: 10px; animation: pulse 1.5s ease-in-out infinite; } @keyframes pulse { 0%, 100% { transform: scale(1); opacity: 1; } 50% { transform: scale(1.2); opacity: 0.7; } }

4️⃣ Dotted Spinner — Dashed Border

Memanfaatkan border-style: dotted yang jarang diketahui. Ketika border dotted pada lingkaran diputar, titik-titik bergerak mengikuti kurva lingkaran — efek yang unik dan berbeda!

CSS — Dotted Spinner .spinner-4 { border: 6px dotted var(--color-4); /* Border titik-titik */ border-radius: 50%; animation: spin 2s linear infinite; /* Lebih lambat = efek lebih jelas */ }

✅ Kesimpulan

  • Semua spinner menggunakan border-radius: 50% + @keyframes spin sebagai fondasi
  • Variasi dicapai dari jenis border (solid, dotted), warna border, dan jenis animasi (rotate vs scale)
  • CSS spinner lebih ringan dari GIF dan bisa disesuaikan ukuran/warnanya dengan mudah
  • Selalu tampilkan spinner saat proses async (fetch data, upload) agar user tahu sistem masih berjalan

✅ Custom Inputs

Checkbox dan Radio Button yang dikustomisasi penuh — tampilan modern, tetap fungsional

✅ Custom Inputs

Checkboxes

Radio Buttons

📖 Mengapa Perlu Custom Inputs?

Tampilan default <input type="checkbox"> dan <input type="radio"> berbeda di setiap browser dan terlihat kuno. Dengan teknik CSS custom inputs, kita menyembunyikan input asli dan menggantinya dengan elemen visual yang sepenuhnya kita kontrol — namun tetap fungsional dan aksesibel.

Perbedaan utama keduanya: Checkbox memungkinkan pilihan lebih dari satu (multi-select), sedangkan Radio Button hanya boleh satu pilihan aktif dalam satu grup (single-select).

🏗️ Struktur HTML: Label Membungkus Input

Kuncinya adalah membungkus <input> asli di dalam <label>. Ini membuat seluruh area label bisa diklik dan otomatis memengaruhi checkbox/radio di dalamnya — tanpa JavaScript sama sekali. Elemen <span class="checkmark"> adalah pengganti visual kita.

HTML — Custom Checkbox & Radio Structure <!-- Custom Checkbox --> <label class="custom-checkbox"> Teks pilihan <input type="checkbox"> <!-- Input asli (akan disembunyikan) --> <span class="checkmark"></span> <!-- Pengganti visual kita --> </label> <!-- Custom Radio (gunakan name yang sama di satu grup) --> <label class="custom-radio"> Pilihan A <input type="radio" name="radio"> <span class="radiomark"></span> </label>

🎨 CSS: Sembunyikan Input Asli, Bangun Ulang Tampilannya

Input asli disembunyikan dengan opacity: 0 (bukan display: none agar tetap bisa diklik dan aksesibel). Lalu .checkmark diposisikan secara absolut di tempat yang sama. Selector input:checked ~ .checkmark mengubah tampilan saat input diceklis.

CSS — Custom Checkbox (Teknik Utama) /* Container label */ .custom-checkbox { display: block; position: relative; padding-left: 35px; /* Beri ruang untuk checkmark */ cursor: pointer; user-select: none; } /* Sembunyikan input asli — tapi JANGAN display:none agar aksesibel */ .custom-checkbox input { position: absolute; opacity: 0; cursor: pointer; } /* Kotak pengganti (unchecked) */ .checkmark { position: absolute; top: 0; left: 0; height: 25px; width: 25px; background: var(--bg-primary); border: 2px solid var(--color-2); border-radius: 5px; transition: all 0.3s; } /* Saat diceklis: background berubah gradient */ .custom-checkbox input:checked ~ .checkmark { background: linear-gradient(135deg, var(--color-2), var(--color-3)); border-color: var(--color-2); } /* Tanda centang (✓) dibuat dari border CSS! */ .checkmark:after { content: ""; position: absolute; display: none; /* Tersembunyi saat unchecked */ left: 7px; top: 3px; width: 6px; height: 12px; border: solid white; border-width: 0 3px 3px 0; transform: rotate(45deg); /* Membentuk tanda ✓ */ } /* Tampilkan tanda centang saat checked */ .custom-checkbox input:checked ~ .checkmark:after { display: block; }

⭕ CSS: Radio Button — Perbedaan dengan Checkbox

Radio button hampir sama strukturnya, hanya berbeda di bentuk (bulat: border-radius: 50%) dan tanda aktifnya berupa lingkaran kecil putih di tengah, bukan tanda centang.

CSS — Custom Radio Button (Perbedaan Kunci) /* Bentuk bulat — ini satu-satunya perbedaan dari checkmark */ .radiomark { border-radius: 50%; /* Lingkaran penuh */ } /* Titik putih di tengah (bukan tanda centang) */ .custom-radio .radiomark:after { top: 6px; left: 6px; width: 9px; height: 9px; border-radius: 50%; /* Titik juga bulat */ background: white; /* Tidak perlu transform: rotate() seperti checkbox */ }

✅ Kesimpulan

  • Gunakan opacity: 0 bukan display: none agar input tetap bisa difokus keyboard (aksesibilitas)
  • Selector input:checked ~ .checkmark adalah inti dari teknik ini — tidak butuh JavaScript sama sekali!
  • Tanda centang (✓) dibuat dari dua sisi border yang dirotasi — tidak menggunakan gambar atau karakter Unicode
  • Checkbox = multi pilihan | Radio = satu pilihan saja (pastikan semua opsi dalam satu grup punya name yang sama)

💬 Tooltips

Teks penjelasan kecil yang muncul saat kursor diarahkan ke elemen — murni CSS!

💬 Tooltips

📖 Apa itu Tooltip?

Tooltip adalah teks keterangan kecil yang muncul saat user mengarahkan kursor ke suatu elemen. Fungsinya memberi konteks tambahan tanpa memenuhi layar — sangat berguna untuk ikon, tombol tanpa teks, atau field formulir yang butuh penjelasan.

Yang luar biasa: tooltip di sini dibuat 100% murni CSS menggunakan pseudo-element ::before dan ::after, tanpa satu baris JavaScript pun!

🏗️ HTML: Satu Atribut, Segalanya

Teks tooltip disimpan langsung di atribut HTML data-tooltip. CSS kemudian mengambilnya menggunakan fungsi attr(). Ini sangat efisien karena tidak perlu membuat elemen tooltip terpisah untuk setiap tombol.

HTML — Tooltip dengan data-attribute <button class="tooltip-trigger" data-tooltip="Teks tooltip kamu di sini" > Hover saya! </button> <!-- Setiap elemen bisa punya tooltip berbeda, cukup ubah nilai data-tooltip-nya --> <button data-tooltip="Simpan dokumen">💾</button> <button data-tooltip="Hapus item">🗑️</button>

🎨 CSS: Pseudo-element ::before dan ::after

Setiap elemen HTML memiliki dua "slot" konten tersembunyi: ::before dan ::after. Di sini ::before digunakan untuk kotak tooltip, dan ::after untuk segitiga kecil (ekor) di bawahnya. Keduanya awalnya opacity: 0 dan muncul saat :hover.

CSS — Tooltip Murni dengan Pseudo-element .tooltip-trigger { position: relative; /* Jadikan referensi posisi untuk tooltip */ } /* ── KOTAK TOOLTIP ── */ .tooltip-trigger::before { content: attr(data-tooltip); /* Ambil teks dari atribut HTML! */ position: absolute; bottom: 120%; /* Posisi di atas tombol */ left: 50%; transform: translateX(-50%); /* Tengahkan secara horizontal */ padding: 8px 12px; background: rgba(0, 0, 0, 0.9); color: white; border-radius: 6px; font-size: 0.9rem; white-space: nowrap; /* Teks tidak wrap */ /* State awal: tersembunyi */ opacity: 0; visibility: hidden; transition: all 0.3s; pointer-events: none; /* Tooltip tidak menghalangi klik */ } /* ── SEGITIGA EKOR ── */ .tooltip-trigger::after { content: ""; position: absolute; bottom: 110%; left: 50%; transform: translateX(-50%); /* Trik border untuk membuat segitiga CSS */ border: 6px solid transparent; border-top-color: rgba(0, 0, 0, 0.9); opacity: 0; visibility: hidden; transition: all 0.3s; } /* ── TAMPILKAN SAAT HOVER ── */ .tooltip-trigger:hover::before, .tooltip-trigger:hover::after { opacity: 1; visibility: visible; }

🔺 Trik Segitiga CSS

Segitiga di CSS dibuat dari border yang runtuh. Saat sebuah elemen berukuran 0×0 dengan border tebal, setiap sudut border membentuk segitiga. Dengan membuat 3 sisi transparent dan satu sisi berwarna, kita mendapat segitiga tunggal!

CSS — Cara Kerja Segitiga dari Border /* Elemen 0×0 dengan border tebal di semua sisi */ .segitiga { width: 0; height: 0; border: 8px solid transparent; /* Semua sisi transparan */ border-top-color: black; /* Hanya atas yang hitam */ /* Hasilnya: ▼ segitiga menghadap bawah */ } /* Untuk tooltip di atas tombol → kita butuh ▼ (ekor menuju tombol) */ /* Maka gunakan border-TOP yang berwarna */

✅ Kesimpulan

  • Simpan teks tooltip di data-tooltip lalu ambil di CSS dengan content: attr(data-tooltip) — tidak perlu JavaScript!
  • ::before untuk kotak tooltip, ::after untuk segitiga ekornya — keduanya pseudo-element yang powerful
  • Tambahkan pointer-events: none pada tooltip agar tidak menghalangi klik user
  • Gunakan tooltip untuk: ikon tanpa label, tombol dengan fungsi tidak jelas, field formulir yang butuh panduan singkat
  • Jangan sembunyikan informasi penting di tooltip saja — tooltip tidak muncul di layar sentuh (mobile)!
